Mac中Hexo博客安装与使用教程

  • blog
  • hexo


Hexo是一款轻量级的使用node.js平台的博客框架，与Jekyll和Octopress类似，使用Markdown来解析文章，但不同于Jekyll和Octopress的是，Hexo的编译速度非常快，据说Octopress微博适量达到50以上编译速度就会慢上几分钟之多，但是Hexo只需要几秒钟就可以了。本文来教大家如何在Mac中搭建Hexo，以及日常如何使用Hexo写博客，希望你看完本篇文章也加入到Hexo大家庭中来！

安装

首先你要确认自己的电脑上必须安装了以下的环境

  • Node.js
  • Git

如果你的电脑已经符合这些条件，你就可以使用npm很简单地安装Hexo:
$ npm install -g hexo-cli

建站

安装 Hexo 完成后，请执行下列命令，Hexo 将会在指定文件夹中新建所需要的文件


新建完成后，指定文件夹的目录如下：

.
├── _config.yml
├── package.json
├── scaffolds
├── scripts
├── source
|   ├── _drafts
|   └── _posts
└── themes

配置

您可以在 _config.yml 中修改大部份的配置。具体可参考Hexo官方网站进行配置，这里就不赘言了。

指令

在日常使用中，你会用到以下指令

1.新建站

$ hexo init [folder]

新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。
例如：

$ hexo init myblog

2.新建文章

$ hexo new [layout] <title>

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。
例如：

$ hexo new "my new post"

3.生成静态文件

$ hexo generate

或者

$ hexo g

在把文章发布到网站之前，我们必须先把网站编译成静态文件，这是必须的操作。

4.本地测试

$ hexo server

你可以使用这个命令来启动本地测试服务器来测试你生成的博客，点击这里测试http://localhost:4000/

5.部署网站

$ hexo deploy

或者

$ hexo d

如果你在_config.yml配置文件中设置了部署环境，你就可以轻松的使用这条命令来把编译好的网站发布到你的远程服务器中。
你也可以使用同时便已部署命令来同时执行generate和deploy命令

$ hexo d -g

* Github部署设置

首先需要安装hexo-deployer-git

$ npm install hexo-deployer-git --save

修改配置文件_config.yml

deploy:
type: git
repo: <repository url>
branch: [branch]   //分支名称，程序会尝试自动检测，可以为空
message: [message] //自定提交信息，可以为空

* Heroku部署设置

首先需要安装hexo-deployer-heroku

$ npm install hexo-deployer-heroku --save

修改配置文件_config.yml

deploy:
type: heroku
repo: <repository url>
message: [message] //自定提交信息，可以为空

主题

创建 Hexo 主题非常容易，您只要在 themes 文件夹内，新增一个任意名称的文件夹，并修改 _config.yml 内的 theme 设定，即可切换主题。一个主题可能会有以下的结构： 

.
├── _config.yml
├── languages
├── layout
├── scripts
└── source

wiki上有很多Hexo主题供你挑选。

写博文

Hexo是使用Markdown语言编写的，具体可参见Markdown语法说明
